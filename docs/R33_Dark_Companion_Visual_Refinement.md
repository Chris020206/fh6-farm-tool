# R33 Dark Companion Visual Refinement

## Purpose

This checkpoint records a reversible visual refinement pass on the PySide6 vertical companion prototype.

The goal was to make the prototype feel more cohesive, premium, and intentionally designed without changing behavior.

It now also records the MVP Visual Completion Sprint and Home Page Implementation Pass that moved the current dark direction toward the approved visual brief.

## Direction

The prototype now explores a restrained dark companion theme.

This direction is intended to support:

- calm focus
- premium utility feel
- stronger visual unity
- reduced default-widget appearance
- clearer card hierarchy
- quieter secondary details
- stronger companion identity

## Refinements Applied

The prototype now uses:

- dark companion canvas
- dark navigation rail and hover overlay
- restrained pink accent treatment
- layered dark utility cards
- recessed secondary cards
- calmer tertiary cards
- clearer commitment treatment for Run
- dark-theme typography colors
- unified shell background, content canvas, and separator treatment
- calmer Automation Environment wording
- Home composed as a launch surface with paired operational signals
- Automation Environment composed as a preparation flow ending in deliberate commitment
- supporting screens moved onto the same card surface system
- top product bar with quiet FH6 Farm Tool identity
- Home ordered as title, opening statement, hero, recommended action, review path, context/status, footer metadata

Home and Automation Environment remain structurally unchanged, but their presentation now feels more composed and less like default Qt scaffolding.

Current composition principle:

> brief-aligned graphite companion surface

Current screen composition:

- Home: ordered launch surface with dominant recommended action
- Automation Environment: preparation flow with deliberate commitment

## Preserved Boundaries

This pass does not introduce:

- navigation behavior changes
- window size changes
- scrolling
- execution wiring
- runner calls
- real input behavior
- automation logic changes
- safety gate changes
- `main.py` changes
- branding or logo work
- new product features

## Reversibility

This is a visual prototype pass only.

If the dark companion direction does not hold up under manual inspection, it can be reverted without affecting automation behavior, product models, safety boundaries, or CLI behavior.

## Current Evaluation Target

Manual inspection should focus on whether the prototype now feels:

- more cohesive
- less default-widget-like
- calm rather than loud
- premium without fake luxury
- focused rather than dashboard-like
- easier to scan at first glance
