# R29 Typography Hierarchy

## Purpose

This checkpoint records typography hierarchy refinement in the vertical companion prototype.

This is not final branding.

This is not final visual design.

## Goal

Make the prototype easier to scan from top to bottom.

The operator should quickly understand:

- where they are
- what the screen is for
- what matters most
- what is secondary
- what can be ignored unless needed

## Refinements Applied

The prototype now uses a lightweight typography hierarchy for:

- screen titles
- philosophy/opening statements
- section and group titles
- summaries
- secondary details
- navigation labels
- footer/status text
- active navigation text

Current hierarchy:

- screen title: `20px`
- opening statement: `13px`
- section title: `13px`
- summary: `12px`
- detail: `11px`
- navigation: `12px`
- footer: `10px`

## Hierarchy Principle

Current principle:

> scan first, detail second

Screen titles should be immediately identifiable.

Summaries should carry the main meaning.

Details should feel quieter and supportive.

Navigation should remain readable without becoming dominant.

## Product Fit

Typography should support:

- calm confidence
- subtle guidance
- premium restraint
- digestibility
- focus-first information density

The prototype should avoid:

- huge dramatic headings
- dense equal-weight text
- fake luxury styling
- gamer-like styling
- corporate dashboard feeling

## Preserved Boundaries

R29 does not introduce:

- navigation behavior changes
- hover overlay changes
- window size changes
- scrolling
- execution wiring
- runner calls
- real input behavior
- automation logic changes
- safety gate changes
- `main.py` changes
- branding or logo system

## Current Status

The prototype should now be easier to scan while remaining restrained.

Further refinement should evaluate:

- Home composition
- Automation Environment vertical readability
- whether secondary details are quiet enough
- whether screen titles are strong enough without becoming dramatic
