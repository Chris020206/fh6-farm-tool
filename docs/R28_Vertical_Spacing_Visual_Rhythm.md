# R28 Vertical Spacing & Visual Rhythm

## Purpose

This checkpoint records compositional spacing refinement in the vertical companion prototype.

This is not a redesign.

This is not final styling.

## Goal

Improve:

- vertical rhythm
- digestibility
- breathing room
- reading flow
- information hierarchy
- calmness
- intentional focus

while preserving:

> compact premium companion identity

## Guiding Principle

Space must feel:

> earned

The prototype should feel restrained but not empty.

The goal is not to fill every visible gap.

The goal is to make each row of focus easier to understand.

## Refinements Applied

The prototype now uses a shared rhythm model for:

- content margins
- header spacing
- section spacing
- group inner margins
- important element spacing
- Home signal rhythm
- Automation Environment section rhythm

Current rhythm values:

- content margin: `18`
- header spacing: `8`
- section spacing: `14`
- group spacing: `10`
- group inner margin: `10`
- important element spacing: `16`

## Preserved Direction

R28 preserves:

- vertical companion layout
- fixed window size
- hover overlay navigation
- stable main content
- no scrolling
- Home as philosophy-first launchpad
- Automation Environment as confidence-to-commitment surface

## Non-Goals

R28 does not introduce:

- navigation behavior changes
- window size changes
- scrolling
- final styling
- execution wiring
- runner calls
- real input behavior
- automation logic changes
- safety gate changes
- `main.py` changes

## Current Status

The prototype should now feel less crowded and more intentionally paced.

Further refinement should evaluate:

- typography hierarchy
- Home composition
- Automation Environment vertical readability
- whether any spacing feels unearned or too sparse
